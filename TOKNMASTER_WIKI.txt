An architectural overview of the ToknNews autonomous AI broadcast system.

1. Overview

ToknNews is an autonomous AI-driven broadcast engine that continuously ingests signals, synthesizes narratives, generates episodes, and distributes content.
The architecture is modular, scalable, and aligned with a “hands-off” operating model where the system autonomously reacts to real-time world activity.

Core Capabilities:

Automated multi-source ingestion (RSS, APIs, on-chain)

Narrative clustering via GPT

Story enrichment + ranking

Timeline orchestration (PDv4)

Media synthesis (voice, visual, CapCut)

Distribution (Metricool)

Backend dashboard + analytics

Secure Control Studio (password + TOTP login)

High-tech unified frontend style system

rate-limit aware backoff

7. Secure Control Studio Login

ToknNews uses a two-factor login system:

Requirements:

Password (bcrypt hash)

TOTP verification (pyotp)

Authenticator enrollment QR code

Endpoints:

/api/studio/qrcode
Returns Base64 QR for enrolling TOTP.

/api/studio/login
Accepts: {password: "...", token: "123456"}
Validates both credentials.

TOTP Storage:

Safe location used:

tokn_studio_totp.json
tokn_studio_password.json

Frontend:

Landing page includes a small hidden login orb button → modal:

Access key (password)

Verification code

Once logged in, user is sent to /studio.html (future no-code studio interface).

9. Known Workflows & Commands
Restart API:
pm2 restart tokn_api --update-env

Restart full stack:
toknctl restart

Ingest now:
curl -X POST https://toknnews.com/api/admin/ingest

Run clustering:
curl -X POST https://toknnews.com/api/admin/analytics/refresh_clusters

GPT diagnostics:
curl https://toknnews.com/api/admin/diagnostics/gpt

Episode generation:
curl -X POST https://toknnews.com/api/admin/generate_episode

10. Development Flow for Future Chats

When starting any future session, the assistant should understand:

ToknNews is composed of:

A Flask backend

A JS + HTML frontend with particle-based UX

A secure studio login requirement

A fully modular ingestion → narrative → render → distribution pipeline

A unified visual language

Active GPT clustering

Long-term story memory via StoryBank

System console microcharts

When modifying code:

Use cat replacement commands

Include cd commands for exact file paths

Never remove working modules

Never rename critical endpoints

Work in the context of how adjacent modules rely on dependencies

Maintain the “ToknNews high-tech holo aesthetic”

When designing new features:

Must align with existing UX

Must expose data in a partner/VC-friendly format

Must support autonomous broadcasting

Should build toward the no-code studio vision
